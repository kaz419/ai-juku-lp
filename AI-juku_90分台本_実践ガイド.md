# AI-juku 90分台本 + 実践ガイド（2テーマ/回）

## 0. 使い方（講師向け）
- 1コマ90分で「2テーマ」
- 各テーマの流れ（35分）：
  - つかみ 3分
  - 例え話 2分
  - なぜAIで可能か 4分
  - 実演 15分
  - 受講者ハンズオン 8分
  - 共有 3分
- 2テーマで70分 + 導入10分 + まとめ10分

---

## 1コマ目（仕事ですぐ効く）
### テーマA：メール自動作成
#### つかみ
「朝、受信トレイが30件。ここで心が折れるか、30分で片付くかで1日の生産性が変わります。」

#### 例え話
「AIは“送信ボタンを押さない秘書”。下書きは秒速、人間は最終判断だけ。」

#### なぜAIで可能？
- 過去の大量文例から、目的別（お礼・催促・断り）パターンを生成
- トーン（丁寧/フランク）を調整可能
- 入力要素（相手、目的、締切）を構造化して文章化

#### 体感価値
- 1通5〜10分 → 30秒〜2分
- 1日20通なら約1.5〜2.5時間削減

#### 実演プロンプト
```
あなたはビジネスアシスタントです。
以下条件で返信メールを作成してください。
- 相手：既存顧客
- 目的：見積送付の連絡
- トーン：丁寧、簡潔
- 入れる要素：納期目安、次アクション、お礼
最後に件名を3案ください。
```

#### ツール候補
- ChatGPT（GPT-4系）
- Gemini
- Claude

---

### テーマB：議事録作成（要点/決定/ToDo）
#### つかみ
「会議で疲れた後の議事録作成が、もう“第2ラウンド”じゃなくなる。」

#### 例え話
「AIは会議の“速記係＋編集者”。人間は中身に集中できる。」

#### なぜAIで可能？
- 文字起こしデータから要点抽出
- 決定事項と未決事項の分類
- タスク（誰が/何を/いつまで）を抽出

#### 体感価値
- 30分会議の議事録：30〜60分 → 5〜10分
- 週5本で2〜4時間削減

#### 実演プロンプト
```
以下の会議メモから議事録を作成してください。
出力形式：
1) 要点（3行）
2) 決定事項
3) 未決事項
4) ToDo（担当/期限つき）
5) 次回会議アジェンダ案
```

#### ツール候補
- Otter / tl;dv / Google Meet transcript + ChatGPT/Gemini/Claude

## 2コマ目（営業導線を自動化）
### テーマA：議事録 → 提案書ドラフト
#### つかみ
「商談後“今日は提案書無理だ…”を、30分で提出できる状態へ。」

#### 例え話
「会話メモを“設計図”に変換する。AIは下書き職人。」

#### なぜAIで可能？
- 会話から課題・期待・制約を抽出
- 提案書の定型構造（課題→解決策→効果→見積）へ整形

#### 体感価値
- 初稿作成 2〜3時間 → 20〜40分

#### 実演プロンプト
```
次の議事録を提案書ドラフトに変換してください。
構成：
- 現状課題
- 提案概要
- 実施ステップ
- 期待効果（定量/定性）
- 概算見積
- スケジュール
```

### テーマB：提案書 → 見積説明文（納得感づくり）
#### つかみ
「値段の勝負ではなく、“納得”の勝負に変える。」

#### 例え話
「見積書は数字、説明文は翻訳。AIは“技術語→顧客語”の通訳。」

#### なぜAIで可能？
- 提案内容と価格項目を対応づけ
- 顧客視点の価値に言い換え

#### 体感価値
- 30〜60分 → 5〜15分

#### 実演プロンプト
```
以下の提案書と見積項目から、顧客向け説明文を作成してください。
条件：
- 専門用語は平易化
- 価格根拠を明確化
- 「なぜ必要か」を一文ずつ示す
- メール本文形式で出力
```

## 3コマ目（分析で意思決定）
### テーマA：売上 × 天気 × 曜日
#### つかみ
「“なぜ売れた/売れない”を勘ではなく、再現可能にする。」

#### 例え話
「売上の健康診断。体温（売上）だけでなく、気温/天候も見る。」

#### なぜAIで可能？
- 複数データの突合
- 相関パターンの説明生成
- 次アクション提案まで自動化

#### 体感価値
- 手分析 半日〜1日 → 10〜30分で仮説

#### 実演プロンプト
```
このCSV（売上）と天気データを分析し、
1) 売上に影響する要因
2) 曜日・天候別の傾向
3) 来週の施策提案（3つ）
を出してください。
```

#### ツール候補
- ChatGPT（Advanced Data Analysis）
- Gemini（表計算連携）
- Claude（CSV分析）

---

### テーマB：広告 × 売上（ROI）
#### つかみ
「広告は“増やす”より“止める判断”が難しい。AIで可視化する。」

#### 例え話
「蛇口が3本ある水道。どの蛇口が利益を生んでるか測る。」

#### なぜAIで可能？
- 媒体別データを統合
- CPA/ROASと売上を同時に解釈

#### 体感価値
- 月次分析資料作成 2〜4時間 → 20〜40分

#### 実演プロンプト
```
媒体別広告データと売上データを統合し、
- 貢献度ランキング
- 継続/停止/改善の判断
- 次月の予算配分案
を提示してください。
```

## GPTs / Gems / その他の使い分け

### 1) GPTs（ChatGPT）
- 向く用途：定型業務（メール、議事録、提案書）を再利用可能にする
- 強み：
  - 指示・口調・フォーマットを固定しやすい
  - 繰り返し業務の品質を揃えやすい
- 例：
  - 「議事録ToDo抽出GPT」
  - 「見積説明文GPT」

### 2) Gems（Gemini）
- 向く用途：Google系業務（Gmail/Docs/Sheets）と相性の良い運用
- 強み：
  - Google Workspace連携前提の現場で導入しやすい
  - 普段の業務導線に載せやすい
- 例：
  - 「営業フォローメールGem」
  - 「週報整形Gem」

### 3) Claude（Projects）
- 向く用途：長文読解、文脈維持、丁寧な要約・構成
- 強み：
  - 長い資料や複数文書の整理が得意

### 4) NotebookLM
- 向く用途：資料群の読解、要点抽出、学習用途
- 強み：
  - 手持ち資料をベースに質疑しやすい

### 5) 自動化連携（Zapier / Make）
- 向く用途：
  - フォーム入力 → 要約 → 通知
  - 会議後 → 議事録整形 → タスク発行
